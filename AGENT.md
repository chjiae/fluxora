# AGENT.md

## 用户可理解的错误提示规范

所有页面和接口必须提供清晰、可理解、可操作的中文错误提示；禁止将技术错误直接展示给用户。

### 禁止展示

页面、弹窗、Toast、表单提示和普通用户错误页不得展示 HTTP 状态码、业务错误编码、Java 异常类名、堆栈、SQL/数据库错误与字段名、后端原始异常消息、内部路径、配置、Token、密码或其他敏感信息。

## 后端接口规范

1. 接口仍返回正确 HTTP 状态码：登录凭证错误、未登录或失效为 `401`；无权限为 `403`；参数校验失败为 `400`；资源不存在为 `404`；内部异常为 `500`。
2. HTTP 状态码与业务错误码仅供程序逻辑使用，不得直接作为用户提示。
3. 错误响应至少包含可识别业务编码、用户可理解中文提示与可选请求追踪标识。
4. 后端不得返回堆栈、SQL、数据库错误、内部实现细节、密码哈希、Token 或配置内容；完整异常只记录在服务端日志。

## 前端错误处理规范

前端必须复用统一错误处理与文案映射，不得直接展示后端原始 `message`、HTTP 状态码、业务编码或异常文本。

| 场景 | 用户提示 |
| --- | --- |
| 用户名或密码错误 | 用户名或密码错误，请重新输入 |
| 账号已停用 | 当前账号已停用，请联系管理员 |
| 所属租户已停用 | 所属租户已停用，暂时无法使用 |
| 所属租户已过期 | 所属租户已到期，请联系管理员续期 |
| 所属租户已删除 | 当前账号所属租户不可用，请联系管理员 |
| 登录状态失效 | 登录已失效，请重新登录 |
| 无页面或操作权限 | 当前账号没有此操作权限 |
| 表单字段不合法 | 在对应字段附近显示具体校验提示 |
| 租户码已存在 | 该租户码已被使用，请更换后重试 |
| 删除受保护自营租户 | 自营租户受保护，无法删除 |
| 停用受保护自营租户 | 自营租户受保护，无法停用 |
| 网络连接失败 | 网络连接失败，请检查网络后重试 |
| 请求超时 | 请求超时，请稍后重试 |
| 服务内部异常 | 服务暂时不可用，请稍后重试 |

## 交互要求

1. 登录失败显示“用户名或密码错误，请重新输入”，不得显示 `401 Unauthorized` 等技术文本。
2. 表单校验优先显示在对应字段附近；非表单失败使用统一 Toast、通知栏、错误页或反馈组件。
3. 提交失败保留用户已填写内容；登录失效提示后安全跳转登录页；无权限展示友好提示并跳转到可访问默认页。
4. 开发环境可输出调试信息到浏览器控制台，生产环境不得向用户界面暴露技术细节。
5. 所有新增页面和接口必须复用统一错误处理机制。

## 验收要求

使用 Playwright 或等价方式验证：错误密码不显示 `401`、异常类名或原始错误；无权限不显示 `403`；表单校验在字段附近展示明确中文提示；网络失败、超时、服务异常使用友好提示；页面、Toast、弹窗和错误页不出现 SQL、堆栈、业务编码或技术错误信息。
