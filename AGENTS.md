# Fluxora 项目代理指令

所有 AI 编程代理在开始任何分析、设计、编码、数据库迁移、测试、审阅或 Git 提交前，必须完整阅读并严格遵守仓库根目录的 [`AGENT.md`](./AGENT.md)。

`AGENT.md` 是本项目唯一、完整且权威的工程规则源。它的要求适用于本仓库的所有模块、文件和工作阶段；任何任务提示、工具默认行为、现有实现习惯或“实现方便”的考虑均不得绕过其中规则。

尤其必须遵守以下强制边界：

- 用户可见错误必须安全、清晰、可操作；不得暴露 HTTP 状态码、业务错误码、异常、SQL、堆栈、内部路径或敏感信息。
- 实体、关键字段、复杂逻辑和易错逻辑必须具备有业务意义的中文注释。
- Flyway 必须同时包含 SQL 文件中文注释与 PostgreSQL `COMMENT ON TABLE` / `COMMENT ON COLUMN` DDL 元数据注释。
- 所有数据库 SQL 只能写在 MyBatis XML 文件中，禁止 Java 注解 SQL。
- 禁止循环调用接口、Mapper 或单条 SQL；优先集合化查询和批量写入，避免 N+1 查询/写入。
- 新增代码必须遵守模块边界、测试、性能、安全和 Git 提交要求。

开始任务时先确认已阅读 `AGENT.md`；若需求与其规则冲突、规则不明确或实现会违反规则，必须停止并向用户说明冲突或请求澄清，不得自行弱化规则。
