# Fluxora 用户、权限与租户管理设计

## 目标

在既有 Fluxora 骨架上交付真实可操作的登录、RBAC、自营租户初始化和平台租户管理。用户按 README 启动已有 PostgreSQL、Redis、平台与前端后，无需手工造数即可完成完整浏览器流程。

## 边界

改动仅限 `fluxora-common`、`fluxora-platform` 和 `fluxora-web`。网关不接入认证、缓存、消息或中继业务。本轮不实现 Provider、模型、API Key、计费、订单、日志和完整用户/角色/权限后台。

## 认证与授权

平台使用 Spring Security。登录成功后签发短期 JWT，写入 `HttpOnly`、`SameSite=Lax` Cookie；前端不保存令牌和密码。每一个受保护请求由后端重新校验用户状态、用户归属租户有效性与权限。

错误响应只提供稳定业务码、安全中文提示与可选追踪标识。前端不展示原始 HTTP 状态、业务码、异常消息或技术细节，统一遵循根目录 `AGENT.md`。

## 数据模型

Flyway 创建 `tenant`、`user_account`、`role`、`permission`、`user_role`、`role_permission` 六张表。所有 SQL 放在 MyBatis XML。

`user_account` 使用明确的 `scope_type`（`PLATFORM` / `TENANT`）及 `tenant_id` 归属约束。一个用户可有多个角色；角色同样拥有作用域，平台角色与租户角色不能跨作用域分配。权限采用稳定字符串编码。

租户包含稳定租户码、名称、类型、启用状态、过期时间、逻辑删除标识与审计字段。有效状态由删除标识、启用状态和过期时间统一计算。`default` / `SELF_OPERATED` 自营租户拥有唯一约束和后端保护：不可删除、停用、过期或修改租户码和类型。

## 系统初始化

Flyway 只管理结构与静态权限/角色基础数据。平台启动时，在事务内幂等初始化系统权限、系统角色及 `admin` 平台管理员。管理员默认初始密码仅用于本地测试，读取安全配置后以 BCrypt 保存；启动不会重置既有密码，也不会记录明文密码。

自营租户由平台管理员首次登录后的后端状态检查触发。两步引导在单个真实后端事务中创建 `default` 自营租户和首个 `TENANT_ADMIN`，由数据库约束保证并发安全与幂等。

## 接口与界面

接口包括登录、当前身份、自营初始化状态/提交和租户分页/详情/新增/编辑/启停/过期设置/删除。后端权限是安全边界。

前端沿用 Vue 路由、Pinia、Axios、现有主题和控制台布局。登录页、初始化向导、权限动态菜单和租户管理页均调用真实接口。租户管理具备分页、搜索、筛选、表单校验、确认操作、响应式呈现及统一友好错误提示。

## 验证

后端针对平台管理员初始化、认证、RBAC、自营初始化、租户 CRUD、状态限制和保护规则提供集成测试。Playwright 运行从登录到租户管理的真实流程，并覆盖桌面、平板和移动端。

## 自检

- 不引入网关业务或无关领域。
- 系统用户和租户用户的作用域有明确持久化模型。
- 自营租户与平台管理员初始化均可重复启动而不重复创建或重置密码。
- 所有对普通用户可见的错误遵循安全中文提示规则。
